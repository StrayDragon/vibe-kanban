## 为什么

当项目规模变大后，很多任务会有相似的标题（例如“重构 X”、“修复 Y”）。仅依赖标题/描述的搜索，
在评审与协作过程中很难快速、精准地定位到某一张卡片。

上游 Vibe Kanban 支持通过短 ID 与数字编号搜索 issue。我们也可以在当前看板中提供同样的“快速定位”能力。

## 变更内容

- 在现有 UUID `id` 之外，对外暴露稳定的数字任务标识 `number` 与短标识 `short_id`。
- 扩展项目看板搜索（Project Kanban search）匹配规则，支持：
  - `#<number>` / `<number>`（精确匹配）
  - `<short_id>` / UUID 前缀（不区分大小写）
  - 现有 title/description 匹配（保持原行为）
- 在看板卡片 UI 中展示数字标识，方便沟通引用（例如 `#123`）。

## 能力

### 新增能力

- `kanban-id-search`：用户可按任务编号或短 ID 搜索/引用任务，而不仅仅依赖标题/描述。

### 变更的能力

<!-- 无 -->

## 影响范围

- 后端 Task 模型 + TS 类型生成（新增 `number` 与 `short_id` 字段）。
- 前端看板过滤逻辑与任务卡片渲染。

## 目标 / 非目标

**目标：**
- 让任务查找更快、更明确。
- 保持向后兼容（仅新增字段，不破坏既有 API）。

**非目标：**
- 本次不做服务端全文检索或索引改造（仍保持客户端过滤）。
- 本次不做按项目维度的重新编号（使用现有稳定标识）。

## 风险

- 暴露内部数字 ID 可能引起困惑 → UI 统一以 `#123` 形式展示，并继续使用 UUID 作为 API 的规范 ID。

## 验证方式

- 创建多个任务，验证看板搜索支持：
  - `#<number>` 与 `<number>`
  - UUID 前缀与 `short_id`
  - title/description（原有行为）
- 运行 `pnpm run generate-types` + `pnpm run check` + `cargo test --workspace`。
